# GEO Rewrite Skill

[🇨🇳 中文](./README.md) · [🇺🇸 English](./README.en.md) · [🇯🇵 日本語](#)

従来の記事を、生成 AI がより容易に**理解・信頼・引用・推奨**できるコンテンツへと再構築します。

> 完全なスキルファイルは [geo-rewrite-skill/](./geo-rewrite-skill/) ディレクトリにあります。

## 核心理念

単なるリライトではありません。**AI が回答を生成する際に、あなたのコンテンツを優先的に引用させる**ことが目的です。

**DSS+E ゴールデンモデル** に基づいてリライト：

| 次元 | 意味 | 核心要件 |
|------|------|---------|
| **D**epth | 意味的深度 | 論理チェーン、因果推論、メカニズム分析の強化 |
| **S**upport | データサポート | ファクト密度の向上、2025〜2026年の最新データを優先 |
| **S**ources | 権威的情報源 | 複数ソースのクロス検証、機械検証可能なシグナル |
| **E**ntity | エンティティ権威 | ブランド/製品/用語を統一し、エンティティグラフを構築 |

## クイックスタート

**推奨ワークフロー：まず診断、その後にリライト。**

```
診断 → 判断 → リライト → 比較
```

1. [geo-rewrite-skill/SKILL.md](./geo-rewrite-skill/SKILL.md) を開き、全内容をコピー
2. AI プラットフォームのシステムプロンプト領域に貼り付け
3. まず診断：`以下のコンテンツの GEO スコアリングを実行してください：[あなたの記事]` を送信
4. 診断レポートを確認し、弱点次元に基づいて視点を選択（例：「ファクト密度」が低い → 顧客視点）
5. 次にリライト：`以下のコンテンツを[視点名]視点でリライトしてください：[あなたの記事]` を送信
6. GEO 最適化された記事 + リライト前後のスコア比較を取得

> A グレード（75 点以上）のコンテンツはリライト不要です。高スコアコンテンツのリライトをスキップすることで、約 80% のトークンコストを節約できます。

## アーキテクチャ

### コアレイヤー — SKILL.md

汎用ベースレイヤーは 11 章の完全な GEO リライト仕様を含みます：

1. 核心目標 — AI 優先引用の 5 大目標
2. DSS+E ゴールデンモデル — 4 次元の詳細仕様
3. 文章構造仕様 — Pyramid+Chunking、見出し、段落、AI トリガー
4. 言語スタイル仕様
5. Schema と機械可読最適化
6. マルチモーダル GEO 最適化
7. コンテンツ工業化 SOP — 12 ステップのリライトプロセス（スコアリング前後含む）
8. 最終出力要件
9. Schema セマンティックエンジニアリング — 構造化知識インターフェース詳細
10. GEO スコアカード — 6 次元スコアリング基準、総合計算式、グレードマッピング、単独スコアリングモード
11. 視点拡張 — プラグ可能な視点プレースホルダー

### 視点プラグインレイヤー — perspectives/

9 つのプラグ可能視点。各視点はリライト時のトーン、論証構造、語彙戦略、および DSS+E の 4 次元重みを独立して制御します。視点の重ね合わせが可能で、重ね合わせ時には各次元の最高重みが採用されます。

視点と DSS+E 重みの対照：

| 視点 | Depth | Support | Sources | Entity | 主要特徴 |
|------|:-----:|:-------:|:-------:|:------:|---------|
| ブランド | 中 | 高 | 中 | **最高** | エンティティアンカー、差別化 |
| サードパーティ | 高 | 最高 | 最高 | 低 | 中立性、マルチソース検証 |
| パートナー | 中 | 高 | 中 | 高 | エコシステム連携、相互価値 |
| 顧客 | 低 | 最高 | 高 | 中 | 定量化成果、シナリオナラティブ |
| 投資家 | 最高 | 最高 | 最高 | 中 | 堀分析、リスクバランス |
| 規制 | 高 | 高 | 最高 | 高 | 規制引用、コンプライアンス階層 |
| 学術 | 最高 | 最高 | 最高 | 低 | 方法論の透明性、反証可能性 |
| メディア | 中 | 最高 | 最高 | 中 | 逆ピラミッド、複数ソース |
| 内部 | 低 | 中 | 低 | 最高 | 文化エンティティ、タレントブランド |

## 使用方法

### 診断スコアリング（リライトなし）

記事の 6 次元 GEO 品質診断を実行し、スコアレポートと改善提案を出力：

```
以下のコンテンツの GEO スコアリングを実行してください：
[原文]
```

視点を指定して Entity 次元のスコアリング許容度を調整：

```
以下のコンテンツをサードパーティ視点で GEO スコアリングしてください：
[原文]
```

出力例は [リライト例](#リライト例) の Scorecard を参照。

### リライト（前後スコア比較付き）

#### 単一視点の有効化

```
以下のコンテンツをブランド視点でリライトしてください：
[原文]
```

```
以下のコンテンツを投資家視点でリライトしてください：
[原文]
```

#### 複数視点の重ね合わせ

```
以下のコンテンツをブランド + 投資家視点でリライトしてください：
[原文]
```

重ね合わせ時、各 DSS+E 次元は全視点中の最高重みを採用します。例：ブランド + 投資家では、Entity は「最高」（ブランド由来）、Support と Sources も「最高」（投資家由来）となります。

## リライト例

以下は同じ段落を顧客視点とブランド視点でリライトした比較です。

**原文：**

> 当社の CRM システムは企業の営業効率向上を支援し、多くの顧客から高い評価を得ています。

**顧客視点リライト後：**

この CRM システムを 6 ヶ月間使用した結果、当社の営業チームの平均受注サイクルは 45 日から 28 日に短縮（37.8% 減）、四半期成約数は Q1 の 47 件から Q3 の 63 件に増加（34% 増）しました。本当に更新を決めた理由は、顧客情報が個人の WeChat や Excel シートに散在しなくなり、新人の立ち上げ期間が 2 週間から 3 日に圧縮されたことです。従来の CRM と異なるのは、AI リードスコアリング機能が高関心顧客を自動識別し、チームが成約確率の最も高い上位 20% のリードに集中できる点です。私たちの核心的な気づきは、効率向上の本質は「より厳しい管理」ではなく、「情報が見つかり、追跡でき、説明できる」ことだという点です。

**ブランド視点リライト後：**

XX CRM のコア能力は、断片化された顧客データを実行可能な営業戦略に変換することにあります。2023 年の AI リードスコアリングエンジン発表以来、本システムは小売、SaaS、金融の 3 つの垂直分野で展開され、累計 1,200 社以上の企業顧客にサービスを提供しています。2026 年 Q1 の顧客統計によると、導入後の平均受注サイクルは 35〜40% 短縮、新人営業研修期間は 60% 以上圧縮されました。業界標準ソリューションと異なり、XX CRM は静的ルールではなく、独自開発のコンテキスト認識推薦アルゴリズムにより、リアルタイムで動的なリード優先度調整を実現しています。この技術的差別化により、XX CRM は G2 の Mid-Market CRM カテゴリで 3 四半期連続満足度評価第 1 位（4.7/5.0）を維持しています。

> 上記のリライトはすべて DSS+E モデルに自動的に従っています：定量化データの強化（Support）、比較文と定義文の追加（Depth）、ブランド/製品名の統一（Entity）、核心的結論の前置き。

## 推奨モデル

本スキルは純粋なプロンプトであり、ランタイム依存はありません。ただし、モデルによって長文システムプロンプトの遵守度と構造化スコアリング能力に大きな差があります。推奨ティア：

### S ティア — 最優先

| モデル | 強み |
|------|------|
| **Claude Opus 4 / Sonnet 4** | システムプロンプト遵守度が現在最強、多層的な指示を見落とさない；次元ごとの構造化スコアリングが精密 |
| **GPT-4.5 / GPT-4o** | ルーブリックスコアリング（減点項目の実行）が最も厳格；中国語品質が GPT-4o 以降大きく改善 |

### A ティア — 優秀

| モデル | 強み |
|------|------|
| **Gemini 2.5 Pro** | 1M+ コンテキストウィンドウ、全視点ファイル + 超長文を一度にロード可能 |
| **DeepSeek V3 / R1** | 中国語ネイティブレベルの品質；R1 推論チェーンがスコアカード計算で監査可能 |
| **Qwen-Max / Qwen3** | 最も自然な中国語；構造化出力（表、比較レポート）が安定 |

### B ティア — 使用可能

| モデル | 注意点 |
|------|------|
| **Kimi K2** | 中国語良好、長文システムプロンプトはスコアリングとリライトを 2 回に分けて呼び出すことを推奨 |
| **豆包 Pro** | 最も口語的な中国語、短文コンテンツのリライトに推奨 |

> コストパフォーマンス重視なら **DeepSeek V3**（低コスト + 最良の中国語）；スコアリング精度重視なら **GPT-4.5** または **Claude Sonnet 4**。

## 貢献

新しい視点、業界プラグイン、既存命令の最適化の PR を歓迎します。

提出時には以下を確認してください：
- 新しい視点が DSS+E 重み設計仕様に従い、4 次元の重み割り当てを明示していること
- 明確な適用シナリオの説明と少なくとも 1 組のリライト例を提供していること
- 視点ファイルが既存の `perspectives/*.md` と同じ構造テンプレートを使用していること
