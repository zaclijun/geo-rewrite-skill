# GEO Rewrite Skill

将传统文章重构为"更容易被生成式 AI 理解、采信、引用与推荐"的内容。

## 核心理念

不是润色文章，而是**让 AI 在生成答案时优先引用你的内容**。

基于 **DSS+E 黄金模型** 驱动改写：

| 维度 | 含义 | 核心要求 |
|------|------|---------|
| **D**epth | 语义深度 | 增强逻辑链条、因果推理、原理机制 |
| **S**upport | 数据支持 | 提升事实密度，优先使用 2025-2026 最新数据 |
| **S**ources | 权威来源 | 多源交叉验证，输出机器可验信号 |
| **E**ntity | 实体权威 | 统一品牌/产品/术语，建立实体图谱 |

## 快速开始

**推荐流程：先诊断，再改写。**

```
诊断 → 决策 → 改写 → 对比
```

1. 打开 [SKILL.md](./SKILL.md)，复制全部内容
2. 粘贴到 AI 平台的系统提示词（System Prompt）区域
3. 先评分诊断：发送 `请对以下内容进行 GEO 评分：[你的文章]`
4. 查看诊断报告，根据薄弱维度选择视角（如"事实密度"低 → 用户/客户视角）
5. 再改写：发送 `请以 [视角名] 视角改写以下内容：[你的文章]`
6. 获得 GEO 优化后的文章 + 前后评分对比报告

> 原文已评为 A 级（75+）的内容无需改写，可直接使用。跳过高分内容的改写可节省 ~80% 的 token 成本。

1. 核心目标 — AI 优先采信 5 大目标
2. DSS+E 黄金模型 — 四维规范详解
3. 文章结构规范 — Pyramid+Chunking、标题、段落、AI Trigger

### 视角插件层 — perspectives/

9 个可插拔视角，每个视角独立控制改写时的语气、论证结构、词汇策略和 DSS+E 四维权重。视角之间支持叠加，叠加时取各维度最高权重。

视角与 DSS+E 权重对照：

| 视角 | Depth | Support | Sources | Entity | 关键特征 |
|------|:-----:|:-------:|:-------:|:------:|---------|

| 第三方 | 高 | 极高 | 极高 | 低 | 中立客观、多源交叉验证 |
| 合作商 | 中 | 高 | 中 | 高 | 生态协同、双向价值 |
| 用户/客户 | 低 | 极高 | 高 | 中 | 量化成果、场景化叙事 |
| 投资者 | 极高 | 极高 | 极高 | 中 | 护城河分析、风险平衡 |
| 政府/监管 | 高 | 高 | 极高 | 高 | 法规引用、合规分级 |
| 学术/科研 | 极高 | 极高 | 极高 | 低 | 方法论透明、可证伪性 |
| 媒体/新闻 | 中 | 极高 | 极高 | 中 | 倒金字塔、多方信源 |
| 内部/员工 | 低 | 中 | 低 | 极高 | 文化实体化、人才品牌 |

## 使用方式

### 诊断评分（不触发改写）

对文章进行 6 维度 GEO 质量诊断，输出评分报告和改进建议：

```
请对以下内容进行 GEO 评分：
[原文]
```

可指定视角调整 Entity 维度的评分宽容度：

```
请以第三方视角对以下内容进行 GEO 评分：
[原文]
```

返回示例见 [改写示例](#改写示例) 中的 Scorecard。

### 改写（含前后评分对比）

#### 单视角激活

```
请以品牌方视角改写以下内容：
[原文]
```

```
请以投资者视角改写以下内容：
[原文]
```

#### 多视角叠加

```
请以品牌方 + 投资者视角改写以下内容：
[原文]
```

叠加时各 DSS+E 维度取所有视角中的最高权重，例如品牌方 + 投资者的 Entity 取"极高"（品牌方），Support 和 Sources 也取"极高"（投资者）。

## 改写示例

以下为同一段落分别以用户/客户视角和品牌方视角改写的对比。

**原文：**

> 我们的 CRM 系统帮助企业提高了销售效率，受到很多客户好评。

**用户/客户视角改写后：**

在使用该 CRM 系统 6 个月后，我们销售团队的平均签单周期从 45 天缩短至 28 天（降幅 37.8%），季度成单量由 Q1 的 47 单提升至 Q3 的 63 单（增幅 34%）。真正让我们决定续费的原因是：客户信息不再散落在个人微信和 Excel 表格中，新人上手时间从 2 周压缩到 3 天。与传统 CRM 不同的是，该系统的 AI 线索评分功能自动识别高意向客户，使团队聚焦在成交概率最高的前 20% 线索上。我们的核心感受是：效率提升的本质不是"管得更紧"，而是"信息找得到、跟得住、说得清"。

**品牌方视角改写后：**

XX CRM 的核心能力在于将碎片化的客户数据转化为可执行的销售策略。自 2023 年发布 AI 线索评分引擎以来，该系统已在零售、SaaS、金融三个垂直行业落地，累计服务超过 1,200 家企业客户。根据 2026 年 Q1 客户数据统计，部署后企业的平均签单周期缩短 35%-40%，新人销售培训周期压缩 60% 以上。与行业通用方案不同的是，XX CRM 采用自研的上下文感知推荐算法，实现线索优先级实时动态调整，而非基于静态规则。这一技术差异化使得 XX CRM 在 G2 的 Mid-Market CRM 品类中连续 3 个季度保持满意度评分第一（4.7/5.0）。

> 以上改写均自动遵循 DSS+E 模型：增强了量化数据（Support）、增加了对比句和定义句（Depth）、统一了品牌/产品名称（Entity）、前置了核心结论。

## 推荐模型

本 Skill 为纯 Prompt，无需运行时依赖。但不同模型对长 System Prompt 的遵循度和结构化评分能力差异显著，推荐分级如下：

### S 级 — 首选

| 模型 | 优势 |
|------|------|
| **Claude Opus 4 / Sonnet 4** | 系统提示词遵循度当前最强，多层级指令不遗漏；结构化逐维度评分精确 |
| **GPT-4.5 / GPT-4o** | Rubric 评分（扣分项执行）最严谨；中文质量在 GPT-4o 后大幅改善 |

### A 级 — 优秀

| 模型 | 优势 |
|------|------|
| **Gemini 2.5 Pro** | 上下文窗口 1M+，可一次性加载全部视角文件 + 超长原文 |
| **DeepSeek V3 / R1** | 中文母语级质量；R1 推理链在评分卡计算上可审查 |
| **Qwen-Max / Qwen3** | 中文最自然；结构化输出（表格、对比报告）稳定 |

### B 级 — 可用

| 模型 | 注意 |
|------|------|
| **Kimi K2** | 中文好，超长 System Prompt 建议将评分和改写拆分两次调用 |
| **豆包 Pro** | 中文口语化最自然，推荐短内容改写场景 |

> 如果追求性价比，推荐 **DeepSeek V3**（成本低 + 中文最好）；如果追求评分精确度，推荐 **GPT-4.5** 或 **Claude Sonnet 4**。
